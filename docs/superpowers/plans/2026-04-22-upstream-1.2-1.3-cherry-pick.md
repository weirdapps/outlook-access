# outlook-cli v1.2.0 + v1.3.0 cherry-pick implementation plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bring upstream `BikS2013/outlook-tool` v1.2.0 + v1.3.0 features into `weirdapps/outlook-access` fork without losing the SharePoint subsystem (`download-sharepoint-link`, `sharepoint-client.ts`, `sharepoint-schema.ts`, `filter-builder.ts`) that exists only in our fork.

**Architecture:** Surgical file extraction from upstream's single squashed commit (`cca2f50`). Pure additions where possible (`get-thread`, `util/dates.ts`, new `OutlookClient` methods). For modified files (`list-mail.ts`, `list-calendar.ts`, `outlook-client.ts`, `cli.ts`, `types.ts`, `config.ts`), patch in new fields/options _alongside_ existing ones — keep our `--since`/`--all` flags as parallel surface (deprecation comes later when callers in `outlook-bridge` + `second-brain` are migrated to `--from`).

**Tech Stack:** TypeScript, vitest, commander.js (CLI), Node 20+, Playwright (login only).
